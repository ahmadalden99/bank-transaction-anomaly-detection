# Design brief

## Product character

This interface is an analytical workbench for reviewing bank-transaction anomaly results. It should feel restrained, practical, evidence-led, and credible. It is a portfolio project, not a fictional SaaS product.

## User priority

1. Understand the current analysis scope.
2. Find transactions that need review.
3. Inspect the evidence behind a risk result.
4. Explore or export the data.
5. Test a new transaction.

## Visual rules

- Use the project name. Do not invent a product brand, logo, or system status.
- Use a light neutral canvas and one navy-blue interaction color.
- Reserve green, amber, orange, and red for risk meaning.
- No gradients, glowing elements, or decorative illustrations.
- No uppercase English eyebrow labels.
- Use 4–6 px corner radii and little to no shadow.
- Use whitespace and typography for hierarchy before adding containers.
- Do not place every section inside a card.
- Prefer sentence case and concise Indonesian copy.
- Prefer tables and familiar bar/line charts over decorative visualizations.
- Show actionable transactions before secondary distributions.
- Make numeric columns easy to compare and use consistent units.

## Anti-patterns

- Purple gradient hero
- Fictional AI/security branding
- Orbital or abstract technology graphics
- Decorative icons without a task meaning
- Pill labels used as decoration
- Excessive cards, rounded corners, and shadows
- Mixed Indonesian and English interface copy
- Color as the only indicator of risk
